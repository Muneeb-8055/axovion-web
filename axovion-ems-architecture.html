System Architecture

Axovion Employee Management System

Attendance tracking, task management, leave system, and overtime — all in one panel with role-based access.

01

Role Hierarchy

3 roles, zero by default

Super Admin

One account. Full control. Creates/deletes employees and admins. Verifies attendance. Manages all permissions.

1 account only

Admin

Zero permissions by default. Super Admin assigns: create employees, approve attendance, approve leaves, assign tasks.

Multiple accounts

Employee

Clock in/out, apply leaves, log task time, see own hours and overtime. Cannot delete or manage others.

N accounts
02

Attendance Rules

208 hrs, 4 leaves, auto-reset

Monthly Target

Target
208 hrs
Working days
26 days
Hours/day
8 hrs

26 days × 8 hrs = 208 hrs/month. Calendar month system. Auto-reset on 1st of every month.

Leave Policy

4 leaves per employee per month. Reset on 1st. Unused leaves lapse — no carry over.

4 leaves/month Half-day = 0.5 Full-day = 1

Advance and same-day leave both allowed. Approved by Admin or Super Admin.

New Employee — Pro-rated Target

When an employee joins mid-month, the system auto-calculates their target based on remaining working days. No manual math needed.

04

Overtime

Tracked separately, never offsets 208

Employee

Tells Admin OT

Admin

Approves / Assigns Project

Admin

Logs Hours in System

Portal

OT Updates Automatically

Pre-approved & Retroactive

Both allowed. Employee can inform Admin before or after doing OT. Admin decides if reason is valid.

Fields Required

Employee, date, hours, reason/project. Reason and project are mandatory for audit trail.

Separate from 208 Target

Overtime does NOT offset the 208 shortfall. They're tracked independently. OT = reward, 208 = requirement.
